# Индивидуальный график смен (hackathon-t2)

Веб-приложение для планирования рабочих смен: сотрудник формирует график на ближайшие две недели, руководитель видит нагрузку по подразделению, сверяет план с фактом и выгружает отчёты. Реализовано на **Python (FastAPI)** с хранением в **SQLite** (или PostgreSQL через переменную окружения).

---

## Возможности

### Авторизация и безопасность

- Вход по **email и паролю**; выдаётся **JWT** (Bearer), срок жизни по умолчанию 7 суток.
- **Регистрация через GUI** (ссылка на экране входа): пользователь выбирает **организацию** и опционально **подразделение**; создаётся роль **сотрудник** с флагом **без входа до подтверждения**. **Руководитель или администратор** той же организации подтверждает заявку на вкладке «Подразделение» или через `POST /api/manager/users/{id}/approve`. После подтверждения пользователь входит как обычно. Пользователи, созданные админом вручную, подтверждены сразу.
- Публичные справочники без токена: `GET /api/public/organizations`, `GET /api/public/teams?organization_id=`.
- Пароли хранятся как **bcrypt**-хеши.
- Поиск пользователя при входе **без учёта регистра** email, лишние пробелы по краям email обрезаются.
- Заголовок для защищённых методов: `Authorization: Bearer <token>`.

### Роли

| Роль | Назначение |
|------|------------|
| **employee** | Свой график: создание, правка и удаление смен (в рамках правил), отправка черновиков на согласование. |
| **manager** | Просмотр смен всей **своей организации**, фильтр по команде; правка плана/факта и статуса; сводка нагрузки по дням; CSV и отчёт план/факт. |
| **admin** | Всё то же для своей организации на уровне API + создание **организаций**, **команд** и **пользователей**. |

Доступ руководителя и администратора к данным **ограничен `organization_id`**: видны только сотрудники той же организации.

### Индивидуальный график (сотрудник)

- Одна смена на **одну дату** на пользователя (уникальность в БД).
- Планирование только на даты от **сегодня** до **сегодня + 14 дней** включительно.
- Поля смены: дата, **план**: время начала и окончания, комментарий; опционально **факт** и статус (для сотрудника подтверждённые смены не редактируются и не удаляются).
- Статусы: **draft** (черновик) → **submitted** (на согласовании) → **confirmed** (подтверждено).
- Массовая операция: перевести **все свои черновики** в `submitted` (`POST .../submit-week`).

### Руководитель подразделения

- Список смен по организации за период, опционально **фильтр `team_id`**.
- **Сводка нагрузки**: число смен по каждому дню в выбранном диапазоне (`/api/shifts/summary`).
- Редактирование любой смены в организации: план, факт, статус, комментарий (`PATCH /api/shifts/org/{id}`).
- **Выгрузка CSV** с разбивкой по дням: дата, сотрудник, email, план/факт начало и конец, статус, комментарий (разделитель `;`, UTF-8 с BOM для Excel).
- **Отчёт план/факт**: суммарные минуты (часы) плана и факта по каждому сотруднику за период (`/api/reports/plan-fact`).

### Организации и команды

- Несколько **организаций**; у каждой — свои пользователи и команды.
- **Команды** с **вложенностью**: поле `parent_id` указывает на родительскую команду (дерево подразделений).
- Администратор создаёт организации, команды и пользователей через **вкладку «Администрирование»** или API (`/api/admin/...`).

### Веб-интерфейс (`static/index.html`)

- Страница входа, **регистрация** (отдельная панель) и хранение токена в `localStorage`.
- Вкладка **«Мой график»**: календарь на **14 дней**, режимы **день / две недели / месяц (обзор)**; модальное окно для создания и редактирования смены; кнопка отправки черновиков на согласование.
- Для **manager** и **admin**: вкладка **«Подразделение»** — блок **заявок на доступ** (подтверждение саморегистрации), диаграмма смен по дням, таблица смен с кнопкой «План/факт», отчёт план/факт, скачивание CSV за выбранный период.
- Для **admin**: вкладка **«Администрирование»** — создание организаций, подразделений и пользователей.

### Интеграция с Google Таблицами

- Заготовка: `GET /api/integrations/google-sheets/status` (роль manager/admin). Сейчас возвращает подсказку по подключению; готовый экспорт — через **CSV** (`/api/export/csv`).

### Данные и запуск «с нуля»

- При первом старте, если в БД **нет пользователей**, выполняется **сид**: демо-организация, иерархия команд (подразделение → команда), три пользователя и одна примерная смена.
- При каждом старте для демо-email вызывается **восстановление пароля**, если текущий хеш не совпадает с демо-паролем (удобно после смены окружения или библиотеки хеширования).
- Файл SQLite по умолчанию: **`schedule.db` в корне проекта** (абсолютный путь), чтобы БД не «терялась» при запуске из другой рабочей папки.

## Технологии

- **FastAPI**, **Uvicorn**
- **SQLAlchemy 2**, **Pydantic Settings**
- **python-jose** (JWT), **bcrypt**
- Фронтенд: один **HTML** + ванильный **JavaScript** (без сборки)

## Переменные окружения

| Переменная | Описание |
|------------|----------|
| `SECRET_KEY` | Секрет подписи JWT; в продакшене задайте длинную случайную строку. |
| `DATABASE_URL` | Строка подключения SQLAlchemy. По умолчанию SQLite в корне репозитория. Пример PostgreSQL: `postgresql+psycopg2://user:pass@host/dbname` |
| `APP_PORT` | Используется скриптом `run.ps1` (порт HTTP-сервера). |

Файл `.env` в корне подхватывается **pydantic-settings** (см. `app/config.py`).

## Установка и локальный запуск

```powershell
cd путь\к\hakaton_t2
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

Рекомендуемый порт **8765** (на **8000** в Windows часто висит **WSL / wslrelay**, из-за чего в браузере открывается не это приложение):

```powershell
.\run.ps1
```

Или вручную:

```powershell
uvicorn app.main:app --host 127.0.0.1 --port 8765 --reload
```

- Приложение: **http://127.0.0.1:8765/**
- Swagger: **http://127.0.0.1:8765/docs**

Используйте **http**, не **https** (TLS в приложении не настроен).

## REST API (краткий справочник)

| Метод | Путь | Кто | Назначение |
|-------|------|-----|------------|
| POST | `/api/auth/login` | — | Вход, JSON: `email`, `password` → JWT |
| POST | `/api/auth/register` | — | Регистрация сотрудника (без JWT): `email`, `full_name`, `password`, `organization_id`, `team_id?` → `{ "message": "..." }` |
| GET | `/api/manager/pending-users` | manager, admin | Список неподтверждённых пользователей своей организации |
| POST | `/api/manager/users/{id}/approve` | manager, admin | Подтвердить доступ |
| GET | `/api/auth/me` | любой авторизованный | Текущий пользователь |
| GET | `/api/public/organizations` | — | Список организаций (для формы регистрации) |
| GET | `/api/public/teams` | — | Команды организации: `organization_id` (обязательный query) |
| GET | `/api/organization` | любой | Организация пользователя |
| GET | `/api/teams` | любой | Список команд организации |
| GET | `/api/shifts/my` | любой | Свои смены (`from_date`, `to_date`) |
| POST | `/api/shifts/my` | любой | Создать смену |
| PATCH | `/api/shifts/my/{id}` | любой | Обновить свою смену |
| DELETE | `/api/shifts/my/{id}` | любой | Удалить смену (не `confirmed`) |
| POST | `/api/shifts/my/submit-week` | любой | Все черновики → `submitted` |
| GET | `/api/shifts/org` | manager, admin | Смены организации (`from_date`, `to_date`, `team_id`) |
| PATCH | `/api/shifts/org/{id}` | manager, admin | План/факт/статус смены |
| GET | `/api/shifts/summary` | manager, admin | Смен на день за период |
| GET | `/api/export/csv` | manager, admin | CSV-файл за период |
| GET | `/api/reports/plan-fact` | manager, admin | Минуты план/факт по сотрудникам |
| POST | `/api/admin/organizations` | admin | Создать организацию |
| POST | `/api/admin/teams` | admin | Создать команду (`organization_id`, `name`, `parent_id`) |
| POST | `/api/admin/users` | admin | Создать пользователя |
| GET | `/api/integrations/google-sheets/status` | manager, admin | Статус заготовки Google Sheets |

## Соответствие требованиям кейса

- Авторизация и ролевая модель (администратор, руководитель, сотрудник).
- Индивидуальный график с окном **2 недели**, хранение в БД.
- Руководитель: распределение смен, нагрузка по дням, план/факт и подтверждение в БД.
- Табличная выгрузка по дням с временем начала и окончания смены (CSV).
- Визуализация по дням / неделям / месяцу в UI; сводка по дням для руководителя.
- Несколько организаций и вложенные команды; публикация через Docker/Uvicorn.

При необходимости доработайте интеграцию с Google Sheets поверх существующего CSV или отдельного фонового задания.
